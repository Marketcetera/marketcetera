Modules that emit & record serialized object data for a data flow from / to a file.

Recorder

  • URN:metc:object:recorder:single
  • Cardinality:Singleton
  • AutoCreated:True
  • AutoStarted:True
  • Function:Data Receiver, Emitter

The module has a customizable output directory location. By default this directory is set to the same value as the value of java.io.tmpdir property. This directory location can be changed at any time. Data received for any new data flows after the change will use the new value. Data received for data flows that existed before the change will continue to go to the old location.

All the data received for each data flow is stored in a file named {output_directory}/flow-{data_flow_id}.obj. The module closes all the files containing the data flow output when it is stopped.

The module will emit all the data that it receives as a part of the data flow if requested. The module will not emit any data if it is the first module in any data flow.

Emitter

  • URN:metc:object:emit:single
  • Cardinality:Singleton
  • AutoCreated:True
  • AutoStarted:True
  • Function:Data Emitter

The module, for each data flow, reads serialized java objects from the file supplied as a request parameter when creating the data flow, and emits each of the read objects into the data flow.

The module cancels the data flow when it reaches the end of file or if it encounters error reading data from the file.
